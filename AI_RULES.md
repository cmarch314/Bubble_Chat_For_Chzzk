# 🤖 AI 작업 규칙 (BubbleChat)

> 이 파일은 AI 에이전트가 작업 시 반드시 따라야 하는 규칙을 정의합니다.
> 세션이 바뀌어도 이 파일을 먼저 읽고 작업을 시작하세요.

---

## 📌 1. Git 커밋 규칙 (가장 중요)

### 1-1. 기능 단위로 반드시 커밋
- **하나의 기능 구현이 완료되면 즉시 `git add` + `git commit`** 을 수행한다.
- 커밋 없이 여러 기능을 연속으로 작업하는 것을 금지한다.
- 세션이 끊기거나 에디터가 재시작되면 미커밋 작업은 소멸할 수 있다.

### 1-2. 커밋 타이밍 기준
| 상황 | 커밋 여부 |
|------|----------|
| 기능 하나 구현 완료 | ✅ 즉시 커밋 |
| 버그 수정 완료 | ✅ 즉시 커밋 |
| 스크래치/테스트 파일 생성 | ⬜ 생략 가능 |
| 단순 조회/분석만 했을 때 | ⬜ 생략 가능 |
| 여러 기능 동시 작업 중 | ✅ 중간 커밋 필수 |

### 1-3. 커밋 메시지 형식
```
feat: [기능명] 한 줄 설명
fix: [버그명] 한 줄 설명
refact: [대상] 한 줄 설명
```

## 📌 2. 작업 전 확인 규칙

### 2-1. 세션 시작 시 반드시 확인
```bash
git log --oneline -5          # 최근 커밋 확인
git status                    # 미커밋 변경사항 확인
git diff --stat               # 변경 파일 목록 확인
```

### 2-2. 이전 미완성 작업 확인
- `git status`에서 **modified** 파일이 있으면, 그 내용이 의도된 변경인지 먼저 확인한다.
- 이전 세션에서 작업하다 커밋 안 된 내용이 있을 수 있으므로 **diff를 확인한 뒤** 작업한다.

---

## 📌 3. 파일 편집 규칙

### 3-1. 문법 검증 필수
- JS 파일 편집 후 반드시 `node -c [파일명]` 으로 문법 오류를 확인한다.
- 오류가 있으면 **커밋 금지**, 수정 후 재확인한다.

### 3-2. 한 번에 너무 많은 변경 금지
- 한 번에 500줄 이상 변경하는 것을 피한다.
- 큰 변경은 **논리적 단위로 쪼개서 순차적으로** 작업한다.
- 각 단위마다 문법 체크 → 커밋.

### 3-3. 한글/인코딩 주의
- PowerShell로 파일을 직접 쓸 때 한글이 깨질 수 있다.
- **가능하면 replace_file_content / multi_replace_file_content 도구**를 사용한다.
- PowerShell로 불가피하게 쓸 경우 UTF-8 BOM 없이 저장: `[System.IO.File]::WriteAllText(path, content, [System.Text.Encoding]::UTF8)`

## 📌 4. 대규모 기능 작업 규칙

### 4-1. 순서
1. **먼저 스펙 파악** - 테스트 파일, 대화 로그, 구현 계획 확인
2. **구현 계획서 작성** - `implementation_plan.md` 업데이트
3. **단계별 구현** - 기능 하나씩, 검증 후 커밋
4. **최종 테스트** - `node scratch/test_*.js` 실행

### 4-2. 작업 추적
- 큰 작업 시작 전 `task.md`에 TODO 목록 작성
- 완료 항목은 `[x]`로 체크

---

## 📌 5. 복구 불가 상황 예방

### 5-1. 금지 행동
- ❌ 기능 구현 후 커밋 없이 다른 기능 작업 시작
- ❌ 파일 전체를 덮어쓰는 방식으로 대규모 수정
- ❌ 기존 동작하던 코드를 삭제하면서 커밋하지 않음
- ❌ 세션 종료 전 `git status` 확인 없이 종료

### 5-2. 작업 종료 전 체크리스트
```
□ node -c 로 문법 오류 없음 확인
□ git add + git commit 완료
□ git log --oneline -3 으로 커밋 확인
□ 미완성 작업은 task.md에 남은 TODO 기록
```

## 📌 6. HuntEffect.js 작업 특이사항

- 파일이 2000줄 이상의 대형 파일이므로 **부분 편집만 허용**
- 전체 덮어쓰기(Overwrite) 절대 금지
- 편집 후 반드시 `node -c js/effects/HuntEffect.js` 실행
- 템플릿 리터럴(백틱) 내부의 HTML은 replace 도구로 편집이 어려울 수 있음
  → PowerShell의 `[System.IO.File]::ReadAllLines` + 라인 번호 기반 교체 사용
